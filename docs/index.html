MemoryDao — 内存存储

使用静态 List 集合存储数据，AtomicInteger 生成自增ID。适合快速测试和功能验证，无需数据库。

// StudentMemoryDao 核心实现
private static final List<Student> data = new ArrayList<>();
private static final AtomicInteger idGenerator = new AtomicInteger(1);

public List<Student> findByPage(int page, int size) {
    int start = (page - 1) * size;
    int end = Math.min(start + size, data.size());
    return new ArrayList<>(data.subList(start, end));
}

DBDao — 数据库存储

基于 JDBC 硬编码 SQL 语句，使用 PreparedStatement 防止 SQL 注入，操作 MySQL 数据库。

DBReflectDao — 反射动态 SQL

通过 DBReflectUtil 读取实体类的 @Entity/@ID/@Column 注解，动态生成 INSERT/UPDATE/DELETE/SELECT SQL，替代硬编码，代码通用性强。

RemoteDao — 远程网络调用

客户端通过 Socket 连接服务端（8888端口），将操作指令和参数封装为 JSON 发送，服务端处理后返回 JSON 响应，实现跨进程数据操作。

// StudentRemoteDao 远程调用
private JSONObject sendRequest(JSONObject request) {
    Socket socket = new Socket(host, port);
    PrintWriter writer = new PrintWriter(socket.getOutputStream());
    writer.println(request.toString());  // 发送JSON请求

    BufferedReader reader = new BufferedReader(
        new InputStreamReader(socket.getInputStream()));
    String line = reader.readLine();    // 读取JSON响应
    return JSONObject.fromObject(line);
}

快速开始
三步运行项目
步骤 1：初始化数据库
  执行 sql/init.sql 创建 stu_manage 数据库及 student、teacher 表

步骤 2：配置 DAO 模式
  修改 web.xml 中 daoType 参数：
    memory     → 内存存储（默认，无需数据库）
    db         → MySQL 数据库
    dbReflect  → 反射动态 SQL
    remote     → 远程 Socket 调用（需先启动 Server）

步骤 3：部署到 Tomcat
  mvn clean package -DskipTests
  将 target/StudentManage.war 放入 Tomcat webapps 目录
  访问 http://localhost:8080/StudentManage
  登录账号：xcy / ddd

远程模式额外步骤：
  先运行 com.stu.server.Server 启动服务端（监听 8888 端口）
  再将 daoType 改为 remote 并重启 Tomcat
